# 2026-04-09 日志

## 今日主要工作

### 1. QMD GPU加速修复（重大突破）
- **问题**：之前记录"CUDA 12.8与node-llama-cpp不兼容"经查证不准确
- **根因**：qmd的llm.js调用`getLlama()`时未传`gpu`参数，导致默认只用CPU
- **验证**：
  - `getLlama({build: 'autoAttempt'})` → GPU: false
  - `getLlama({gpu: 'auto', build: 'autoAttempt'})` → GPU: cuda ✅
- **修复**：patch了qmd的llm.js，添加QMD_GPU环境变量支持（默认auto）
- **结果**：qmd status现在显示 `GPU: cuda (offloading: yes)`，RTX 4070可用

### 2. 调研过程
- 使用Tavily搜索node-llama-cpp CUDA兼容性问题
- 查阅官方文档：prebuilt binaries支持CUDA 12.4+和13.1+
- 发现GitHub issue #580：Windows VMM crash问题
- 实测验证：`npx node-llama-cpp inspect gpu`显示CUDA available
- 编写test-llama-gpu.mjs对比不同配置的GPU检测结果

## 技术要点
- node-llama-cpp的`getLlama()`需要显式传`gpu: 'auto'`或`gpu: 'cuda'`才能启用GPU
- `build: 'autoAttempt'`只控制是否从源码编译，不控制GPU选择
- CUDA 12.8完全兼容node-llama-cpp 3.18.1的预编译二进制
