## 一、data-analyst-pro（数据分析）

### 核心能力
- 完成用户委托的数据分析任务
- 处理CSV/Excel等文件
- 生成分析报告和可视化

### 使用方法

- 用文件名直接访问（如 `open('data.csv', 'r')`）

- 趋势统计
- 竞品数据对比

## 二、rag（知识库系统）

### 核心能力
- 构建RAG检索增强生成系统
- 向量数据库选择和部署
- 分块策略优化
- 检索调优（top-k、混合搜索、重排序）
- 评估指标构建
- 生产环境监控

### 决策清单（构建前必问）
- [ ] 文档类型和数量？
- [ ] 延迟要求（实时聊天vs批量）？
- [ ] 更新频率（文档多久变一次）？
- [ ] 访问控制需求（谁能看什么）？
- [ ] 合规要求（GDPR/HIPAA/SOC2）？
- [ ] 预算（托管vs自建）？

### 关键规则
- **永远不要跳过访问控制** — 检索时过滤，不要事后过滤
- **始终重叠分块** — 10-20%防止边界上下文丢失
- **先评估再优化** — 先建测试集，再调优
- **同一嵌入模型** — 查询和文档必须用相同的模型
- **监控相似度分数** — 平均分下降说明有问题

### 常见失败模式
| 症状 | 原因 | 修复 |
|------|------|------|
| 检索到错误文档 | 查询太模糊、分块差 | 扩展查询、更小分块 |
| 相关文档漏检 | 未索引、相似度低 | 检查摄入、混合搜索 |
| 答案幻觉 | 上下文太短 | 增加top-k、重排序 |
| 响应慢 | 大分块、无缓存 | 优化分块大小、缓存嵌入 |
| 结果不稳定 | 重排序不确定 | 设种子、用稳定排序 |

- `implementation.md` — 实现模式与代码
- `evaluation.md` — 评估指标与调试

## 三、douyin-video-downloader（抖音下载）

### 核心能力
- 抖音视频无水印下载
- 通过第三方解析服务绕过反爬

### 下载方法

**方法1：在线解析**
1. 打开 https://douyin.iiilab.com/
2. 粘贴视频链接
3. 点击解析
4. 获取无水印下载链接

**方法2：自动化脚本**
```bash
curl -L "视频下载地址" -o "输出文件名.mp4"
```

### 技术原理
- 使用第三方解析服务获取视频源
- 返回无水印CDN地址（douyinvod.com域名）

### 失败原因
- 登录状态限制（部分视频需要登录）
- URL时效性（通常几分钟到几小时）
- IP限制（频繁请求可能被封）

### 解决方案
- 手动下载（App或网页端）
- 更换解析服务（snapany.com、tikmate.online）
- 使用已登录浏览器的Cookie
