## 2026-04-04 CUDA问题调研记录

### 问题现象
- `qmd embed` 报错：`ggml-cuda.cu:98: CUDA error`
- GPU状态正常（RTX 4060, 7.5GB空闲）
- CUDA版本：13.2

### 调研尝试
1. ✅ 重装qmd - 无效
2. ✅ 清除npm cache - 无效
3. ❌ 尝试环境变量禁用CUDA - 无效
4. ❌ 查看源码 - D:\a\node-llama-cpp目录不存在（可能是临时目录或缓存）
5. ❌ 尝试手动构建 - 缺少build.js脚本

### 根因分析
node-llama-cpp使用的是预编译二进制，CUDA支持在编译时嵌入。CUDA 13.2相对较新，node-llama-cpp 3.18.1可能：
1. 预编译的CUDA版本不支持13.2
2. 或CUDA驱动初始化时版本检测失败

### 可能的解决方案
1. **等待更新**：node-llama-cpp新版本可能修复
2. **降级CUDA**：安装CUDA 12.x驱动
3. **强制CPU模式**：修改node-llama-cpp源码
4. **使用Bun运行时**：Bun可能处理方式不同

### ✅ 解决方案（2026-04-04）
**使用Bun代替Node.js运行qmd**：
```bash
bunx @tobilu/qmd embed
```

**结果**：
- ✅ 成功嵌入22个chunks，10个文档
- ✅ 向量总数：702（之前680 + 新增22）
- ✅ 语义搜索正常

**原理**：Bun运行时对CUDA的处理方式与Node.js不同，可能绕过了版本检测问题。
